# card/login-feature [L3] 결과

## 꼭 필요해 보이는 기능

1. 상태 저장/복원 안정화
   화면 회전이나 프로세스 재생성 시 카드 스택 위치를 더 안전하게 복원할 필요가 있다.
2. 접근성 개선
   스와이프 중심 UI라 TalkBack, 키보드, 대체 입력 지원이 약하다.
3. 대용량 데이터 성능 검증
   페이징은 언급되지만 실제 대량 카드 목록에서의 성능 가이드가 더 필요하다.
4. 테스트 커버리지 추가
   레이아웃 매니저와 상태 전이 로직은 회귀 위험이 높다.
5. Compose 등 현대 UI 스택 연계 전략
   현재 RecyclerView 중심이라 최신 Android 생태계 연결점이 필요하다.

## 있으면 좋을 기능

1. 카드 상태 복원 API
   중간 위치 저장 후 복원하는 공식 API가 있으면 사용성이 좋아진다.
2. 제스처 커스터마이징 확장
   속도, 마찰, 축별 제한을 세밀하게 조정할 수 있으면 활용 범위가 넓어진다.
3. 애니메이션 프리셋 제공
   기본 세팅만으로도 다양한 UX 스타일을 쉽게 재현할 수 있다.
4. 샘플 앱 시나리오 확장
   Tinder형 외에 상품 추천, 작업 큐 같은 예제가 추가되면 도입 판단이 쉬워진다.
5. 공개 성능 가이드 문서
   아이템 복잡도, 이미지 로딩, `notifyDataSetChanged` 회피 전략을 정리할 수 있다.

## 없어도 되지만 좋은 인사이트가 있는 기능

1. 스와이프 히트맵 시각화
   사용자가 어느 방향으로 얼마나 자주 넘기는지 UX 분석에 도움이 된다.
2. 디버그 오버레이
   현재 topPosition, ratio, direction을 실시간으로 보면 라이브러리 학습에 좋다.
3. 샘플용 애니메이션 비교 모드
   같은 데이터로 여러 설정값을 비교할 수 있다.
4. 이벤트 타임라인 로거
   `onCardDragging`, `onCardSwiped` 호출 순서를 쉽게 검증할 수 있다.
5. 디자인 시스템 연동 예제
   Material 3 기반 샘플이 있으면 실제 앱 적용성이 높아진다.
