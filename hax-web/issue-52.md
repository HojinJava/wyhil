# hax-web/login-feature [L3] 결과

## 꼭 필요해 보이는 기능

1. 인증/인가 흐름 명세화
   메뉴 권한, 사용자 권한, 세션 정책이 분산되어 있어 기준 문서가 필요하다.
2. 로그인 실패 처리 표준화
   `loginFail` 같은 예외 코드를 사용자 메시지, 로그, 감사 이력과 연결해야 한다.
3. 사용자/권한 변경 회귀 테스트
   권한 트리와 사용자별 접근 제어는 자동 테스트가 없으면 쉽게 깨진다.
4. 세션 및 만료 정책 점검
   Spring Session 사용 여부와 실제 세션 저장 전략을 운영 기준으로 재검토해야 한다.
5. 인증 진입점 정리
   로그인 URL, 메일 내 로그인 링크, 메뉴 접근 API의 경계를 명확히 해야 한다.

## 있으면 좋을 기능

1. 관리자용 로그인 감사 화면
   실패율, 잠금 여부, 최근 로그인 기록을 운영자가 볼 수 있으면 대응이 빨라진다.
2. 비밀번호 정책 강화 옵션
   만료 주기, 최근 비밀번호 재사용 금지 같은 확장 포인트가 유용하다.
3. 비정상 접근 탐지
   동일 계정의 다중 IP 접근, 짧은 시간 내 반복 실패를 감지할 수 있다.
4. SSO 연계 준비
   사내 계정 체계와 묶일 가능성이 있으면 인터페이스를 미리 정리하는 편이 낫다.
5. 인증 관련 API 문서 보강
   Swagger에 보안 전제와 실패 응답 케이스를 더 구체적으로 적을 수 있다.

## 없어도 되지만 좋은 인사이트가 있는 기능

1. 권한 트리 시각화 도구
   복잡한 메뉴 권한을 운영자가 이해하기 쉬워진다.
2. 로그인 성능 프로파일링
   DB 조회, 암호 검증, 세션 저장 구간별 병목을 숫자로 확인할 수 있다.
3. 메일 링크 로그인 추적
   사용자 메일 발송 후 실제 로그인 전환률을 확인할 수 있다.
4. 기능별 보안 등급 표
   어떤 메뉴가 고위험인지 분류해 인증 정책을 다르게 줄 수 있다.
5. 샘플 시나리오 기반 QA 체크리스트
   신규 운영자나 QA가 로그인 흐름을 빠르게 검증할 수 있다.
