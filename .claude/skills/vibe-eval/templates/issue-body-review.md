## 발견된 문제

### CRITICAL

없음

---

### HIGH

**H1. PR 라벨 예시가 실제 라벨명과 불일치**

현재: `(예: claude, wyhill, cortex-code)`
실제 GitHub 라벨: `model:Claude`, `model:Wyhill`, `model:코덱스 코드`

AI가 예시를 그대로 따르면 존재하지 않는 라벨을 사용하게 되어 PR 생성이 실패하거나 라벨 없이 제출됨.

수정 제안: `(예: model:Claude, model:Wyhill, model:코덱스 코드)`

---

**H2. 실행 순서와 squash 규칙이 연결되지 않음**

실행 순서 3번: "변경사항을 본인 브랜치에 커밋합니다"
규칙 표: "커밋 1개로 PR 제출 (squash 필수)"

두 내용이 분리되어 있어 AI가 여러 커밋으로 개발한 뒤 squash 없이 PR을 제출할 수 있음.

수정 제안: 실행 순서 3번을 "변경사항을 단일 커밋으로 squash하여 본인 브랜치에 커밋합니다"로 변경.

---

### MEDIUM

**M1. 토큰 수 측정 기준 없음**

`| 토큰 수 | |` 항목만 있고 어떻게 측정하는지 기준이 없음. 모델마다 토큰 카운트 방식이 다르고, 세션 전체인지 출력만인지도 불명확하여 모델 간 비교가 불가능해짐.

수정 제안: 예시 추가 — `| 토큰 수 | (예: 입력 12,000 / 출력 3,500) |` 또는 측정 기준을 주석으로 명시.

---

**M2. 자체 완성도 채점 기준 없음**

`| 자체 완성도 | /5 |` 항목에 1~5 각 점수가 무엇을 의미하는지 루브릭이 없음. AI마다 다른 기준으로 자가 평가하게 됨.

수정 제안: 간단한 루브릭 추가.
예시:
- 5: 추가 프롬프트·수동 수정 없이 요구사항 100% 충족
- 4: 미세한 수동 수정 1회 이내
- 3: 추가 프롬프트 1~2회 필요
- 2: 요구사항 일부 누락
- 1: 요구사항 대부분 미충족

---

**M3. 리뷰어가 평가할 PR 범위 불명확**

"지정받은 PR을 읽고"라고 되어 있으나 어떤 PR을 지정받는지, 몇 개를 평가해야 하는지 명시 없음. 리뷰어가 이슈만 보면 평가 대상을 알 수 없음.

수정 제안: "이슈에 연결된 PR 목록에서 지정받은 PR 1개를 평가합니다" 등 범위 명시.

---

**M4. 평가 항목 "코드 품질"이 비코드 태스크에 부적합**

기능 리스트업, 문서 작성 등 코드가 없는 태스크에도 "코드 품질" 항목이 그대로 적용됨. 리뷰어가 해석하기 모호함.

수정 제안: "결과물 품질 (코드 가독성·구조·예외처리 또는 문서 완성도·구조)"처럼 태스크 유형을 포괄하는 표현으로 변경.

---

### LOW

**L1. 플레이스홀더 치환 주체가 명시되지 않음**

`{SLUG}`, `{FEATURE_DEF_URL}`, `{ISSUE_NUMBER}`, `{MODEL_RULES_TABLE}` 등의 플레이스홀더가 스킬에 의해 자동 치환되는지, 수동으로 채워야 하는지 템플릿 자체에 설명이 없음. 템플릿을 직접 읽는 사람이나 다른 AI가 혼동할 수 있음.

수정 제안: 파일 최상단에 주석 형태로 치환 주체 명시 (예: `w-create-issue 스킬이 자동 치환합니다`).

---

**L2. "추가 프롬프트" 정의 없음**

`| 추가 프롬프트 | 없으면 "없음" |` 항목에서 "추가 프롬프트"가 무엇인지 정의 없음. AI의 질문 요청도 포함인지, 사람이 보낸 보완 지시만 해당하는지 불명확.

수정 제안: "이슈 외 별도로 입력한 프롬프트 횟수 및 내용"으로 구체화.
