# mall/create-codebase [L3]

## 공통 프롬프트

**중요: 로컬 작업 워크플로우 지시사항:**
나는 지금 로컬에서 작업 중입니다. 전체 코드베이스가 이 채팅/워크스페이스에 제공되어 있습니다.
- 만약 워크스페이스 전체를 볼 수 있다면 (Cursor, Claude Projects, Claude Desktop, Windsurf 등), 프로젝트의 **모든 파일**을 빠짐없이 분석하세요.
- zip 파일을 업로드한 경우, 압축을 풀고 전체 프로젝트 구조를 완전히 분석하세요.
- 어떤 폴더도 가정하거나 건너뛰지 마세요. 코드베이스 전체에 대한 완전하고 철저한 분석을 원합니다.
- 컨텍스트가 너무 커서 한 번에 처리하기 어렵다면, 먼저 특정 폴더를 제공해 달라고 요청하세요. 하지만 가능한 모든 것을 분석하는 것을 최우선으로 하세요.

너는 15년 이상 경력의 엘리트 소프트웨어 아키텍트이자 시니어 풀스택 엔지니어다.

제공된 전체 코드를 깊이 분석해서 **codebase.md** 파일을 전문적으로 작성해줘.

**절대 조건:**
- 출력은 `codebase.md` 파일 내용 **전체만** Markdown으로 출력한다. 다른 설명, 인사, "여기 codebase.md 내용입니다" 같은 말은 절대 붙이지 마라.
- Markdown 형식은 깔끔하고 전문적으로, 테이블·코드블록·Mermaid 다이어그램을 적절히 사용한다.
- 매우 상세하고 객관적이며 실무적인 인사이트를 포함한다.

**codebase.md 구조 (반드시 아래 순서와 제목으로 작성):**

1. **프로젝트 개요**
2. **기술 스택 및 의존성**
3. **프로젝트 구조**
4. **핵심 아키텍처**
5. **주요 파일 및 모듈 분석**
6. **코드 품질 분석**
7. **개선 로드맵**
8. **개발 가이드**
9. **AI 어시스턴트 참고 섹션** (미래 AI가 이 코드를 다룰 때 알아야 할 핵심 포인트 정리)

마지막으로 **"AI Assistant Reference Section"**은 꼭 포함해서, 이후 AI가 이 프로젝트를 빠르게 이해할 수 있게 만들어줘.

출력은 `# 프로젝트명 - Codebase Documentation` 제목으로 시작해서 전체 Markdown 내용만 출력해.
