# supersocket/login-feature [L3] 결과

## 꼭 필요해 보이는 기능

1. 인증/권한 처리 추가
   현재 예제는 접속 후 바로 패킷을 주고받기 때문에 운영 환경 기준의 사용자 식별 계층이 없다.
2. 패킷 검증 및 예외 응답 표준화
   잘못된 `PacketID`, 길이 불일치, 손상된 body에 대한 공통 에러 코드가 필요하다.
3. 연결 상태 모니터링
   세션 수, 포트별 접속 수, 최근 에러를 UI나 로그에서 즉시 확인할 수 있어야 한다.
4. 요청/응답 프로토콜 문서화
   현재는 코드와 발표 자료 중심이라 외부 클라이언트가 구현하기 어렵다.
5. 테스트 가능한 서버 코어 분리
   WinForms와 서버 로직 결합이 강해서 자동화 테스트가 어렵다.

## 있으면 좋을 기능

1. 헬스 체크 및 메트릭 노출
   포트별 상태를 외부 모니터링에서 읽을 수 있으면 운영성이 좋아진다.
2. 패킷 리플레이 도구
   특정 요청을 재현해 디버깅하는 기능이 있으면 장애 분석 속도가 빨라진다.
3. TLS 또는 내부망 암호화 옵션
   예제 성격을 넘어 실사용으로 확장할 때 필요하다.
4. 설정 파일 기반 포트/핸들러 등록
   현재는 코드 수정 중심이라 배포 유연성이 낮다.
5. 부하 테스트 시나리오 샘플
   단일 에코 패킷 외에 동시 접속 시나리오가 있으면 품질 검증에 유리하다.

## 없어도 되지만 좋은 인사이트가 있는 기능

1. 포트별 트래픽 비교 대시보드
   멀티포트 예제의 차이를 시각적으로 보여주는 데 유용하다.
2. 패킷 구조 시각화 도구
   헤더/바디를 UI에서 해석해 보여주면 교육용 가치가 커진다.
3. 로그 상관관계 ID
   요청 단위 추적을 넣으면 예제 이상으로 확장할 때 좋은 기반이 된다.
4. 코드 생성형 패킷 DTO
   수동 바이트 조립을 줄이면 유지보수성이 좋아진다.
5. 플랫폼 독립 실행 예제
   현재 WinForms 비중이 커서 콘솔/서비스 버전이 있으면 학습 범위가 넓어진다.
