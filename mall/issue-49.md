# mall/login-feature [L3] 결과

## 꼭 필요해 보이는 기능

1. 로그인 이벤트 감사 로그 고도화
   관리자 로그인은 존재하지만 IP, 디바이스, 실패 사유를 더 구조적으로 남길 필요가 있다.
2. 관리자/사용자 로그인 정책 분리
   `mall-admin`과 `mall-portal`의 인증 경계가 달라 운영 정책도 분리되어야 한다.
3. 토큰 만료 및 갱신 UX 개선
   장시간 사용 시 재로그인 부담을 줄이는 흐름이 필요하다.
4. 계정 잠금 및 이상 로그인 탐지
   반복 실패, 지역 변경, 비정상 시간대 접근을 탐지해야 한다.
5. 인증 관련 통합 테스트
   관리자 로그인, 권한 조회, 토큰 검증 경로를 회귀 테스트로 고정할 필요가 있다.

## 있으면 좋을 기능

1. MFA 확장 포인트
   SMS 또는 이메일 OTP를 붙일 수 있는 구조가 있으면 보안 확장성이 좋다.
2. 최근 로그인 이력 조회 API
   계정 보안 점검과 고객 문의 대응에 도움이 된다.
3. 관리자 권한별 세션 정책
   고권한 계정에 더 짧은 세션이나 추가 검증을 줄 수 있다.
4. SSO 또는 외부 IdP 연동 준비
   B2B 확장 시 자체 로그인만으로는 한계가 있다.
5. 인증 실패 사유의 사용자 친화적 메시지
   보안을 해치지 않는 선에서 운영/사용자 모두 이해 가능한 응답이 필요하다.

## 없어도 되지만 좋은 인사이트가 있는 기능

1. 로그인 후 추천 초기화 전략 분석
   인증 직후 사용자 상태를 어떤 캐시와 동기화할지 정리할 수 있다.
2. 권한 변경 시 토큰 무효화 전략 문서화
   보안 사고 대응 체계를 명확히 할 수 있다.
3. 인증 경로 성능 벤치마크
   Redis, DB, JWT 처리 비용을 수치로 확인하면 튜닝 포인트가 보인다.
4. 운영 콘솔용 인증 상태 대시보드
   실패율, 잠금 계정, 활성 세션을 한 번에 볼 수 있다.
5. 인증 관련 데이터 보존 정책 정리
   로그 저장 기간과 개인정보 최소화 기준을 명확히 할 수 있다.
