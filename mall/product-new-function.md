# mall 프로젝트 추가 기능 리스트업

현재 mall 프로젝트의 기능을 분석하여 추가할 수 있는 기능을 3개의 카테고리로 분류합니다.

현재 구현된 주요 기능: 상품 관리(PMS), 주문 관리(OMS), 판촉 관리(SMS), 콘텐츠 관리(CMS), 회원 관리(UMS), Elasticsearch 검색, 알리페이 결제, 장바구니, 쿠폰, 플래시세일, 위시리스트, 열람 이력

---

## 1. 무조건 필요해 보이는 기능

| 기능 | 설명 |
|------|------|
| 상품 리뷰 / 평점 시스템 | 구매 완료 후 별점과 텍스트 리뷰를 남기는 기능. 현재 프로젝트에 리뷰 도메인이 전혀 없어 신뢰도 지표가 부재함 |
| 소셜 로그인 | 카카오·네이버·구글 OAuth2 연동. 현재 자체 계정 + JWT 방식만 지원하며 간편 가입 경로가 없음 |
| 배송 추적 | 주문 상태(배송중·배송완료)와 운송장 번호를 연계한 실시간 배송 위치 조회. OMS에 주문 상태는 있으나 외부 택배 API 연동이 없음 |
| 재입고 알림 | 품절 상품의 재입고 시 이메일·푸시 알림을 발송하는 기능. 재고(SkuStock) 테이블은 있으나 알림 트리거가 없음 |
| 상품 Q&A 게시판 | 구매 전 고객이 상품에 대한 질문을 남기고 판매자가 답변하는 기능. 현재 CS 채널이 별도로 없음 |

---

## 2. 있으면 좋아보이는 기능

| 기능 | 설명 |
|------|------|
| AI 개인화 추천 | 열람 이력(MongoDB)과 구매 이력을 기반으로 협업 필터링 또는 LLM 임베딩을 활용해 개인화 상품을 추천 |
| 포인트 / 리워드 시스템 | 구매·리뷰 작성·출석 등 활동에 포인트를 부여하고 결제 시 사용하는 기능. 현재는 쿠폰만 존재 |
| 상품 비교 | 동일 카테고리 내 상품 2~4개를 선택해 스펙을 나란히 비교하는 기능 |
| 위시리스트 공유 | 저장된 위시리스트(관심 상품)를 URL 링크로 공유하는 기능 |
| 구독 / 정기 배송 | 소모품류 상품을 주 단위·월 단위로 자동 재주문하는 기능 |

---

## 3. 필요 없으나 있으면 재밌을 것 같은 기능

| 기능 | 설명 |
|------|------|
| 라이브 쇼핑 | 판매자가 실시간 스트리밍으로 상품을 소개하고 시청자가 즉시 구매할 수 있는 기능 |
| 구매 게임화 | 첫 구매·10회 구매 등 조건 달성 시 배지와 칭호를 부여하는 도전과제 시스템 |
| 상품 AR 미리보기 | 카메라를 통해 가구·의류 등을 실제 공간이나 착용한 모습으로 미리 확인하는 기능 |
| 경매 / 역경매 | 한정판 상품을 경매 방식으로 판매하거나 구매자가 희망 가격을 제시하는 역경매 기능 |
