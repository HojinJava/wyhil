# wyhil 프로젝트 지침

## 이 프로젝트에 대해

이 레포지토리는 여러 AI 모델(Claude, Wyhill, Antigravity, Codex 등)이 동일한 태스크를 수행하고 결과를 비교하는 Vibe Eval 플랫폼이다.
`.claude/` 하위의 템플릿, 스킬, 이슈 정의 파일은 Claude만 읽는 게 아니라 **참여하는 모든 모델의 공통 지침**으로 사용된다.

## 이슈 템플릿 수정 시 절대 원칙

`issue-body.md`, `issue-task.md`, `session.md` 등 템플릿 파일을 수정할 때:

- 수정은 **템플릿 파일 자체에만** 적용한다
- 이미 각 모델 브랜치에 배포된 `task/**/issue-*.md` 파일과 기존 GitHub 이슈는 **절대 수정하지 않는다**
- 변경 사항은 이후 새로 생성되는 이슈부터 반영된다

## 지침서·템플릿 수정 원칙

이 프로젝트의 템플릿이나 지침을 수정할 때는 다음을 전제한다:

- 작성한 내용을 읽는 독자가 Claude가 아닐 수 있다
- 특정 모델의 고유 개념(파일 경로, 도구명, API 용어 등)을 예시로 쓰지 않는다
- 모든 AI 모델이 공통으로 이해할 수 있는 언어로 작성한다
- 기능이나 동작을 설명할 때 구현 방식이 아닌 **의도**를 기준으로 서술한다
