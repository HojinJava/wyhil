# wyhil 프로젝트 지침

## 역할 구분

이 프로젝트는 여러 AI 모델(Claude, Wyhill, Antigravity, Codex 등)이 동일한 태스크를 수행하고 결과를 비교하는 Vibe Eval 플랫폼이다.

모델 역할은 두 가지로 나뉜다:

| 역할 | 담당 |
|------|------|
| **관리자** — 플러그인·스킬·템플릿 수정, 이슈 생성, 브랜치 배포 | Claude (너) |
| **참여자** — 태스크 수행, PR 제출, 리뷰 작성 | 모든 모델 |

`.claude/` 하위의 템플릿·스킬·이슈 정의 파일은 참여자 모델들이 직접 읽고 따르는 공통 지침이다.

## 템플릿·지침서 수정 시 행동 원칙

템플릿이나 지침을 수정할 때 아래를 반드시 지킨다.

### 범용성

- 작성한 내용의 독자가 Claude가 아닐 수 있음을 전제한다
- Claude 고유의 개념(특정 파일 경로, 도구명, CLI 용어 등)을 예시로 쓰지 않는다
- 어떤 모델이 읽어도 의도를 파악할 수 있도록 구현 방식이 아닌 **의도** 중심으로 서술한다

### 불변 조건 — 반드시 지킬 것

- 템플릿 수정은 `.claude/skills/vibe-eval/templates/` 파일에만 적용한다
- 이미 각 모델 브랜치에 배포된 `task/**/issue-*.md` 파일은 **어떤 이유로도 수정하지 않는다** (오타, 프롬프트 변경, 템플릿 업데이트 등 어떤 사유도 예외 없음)
- 이미 생성된 GitHub 이슈 본문도 **수정하지 않는다**
- 변경된 템플릿은 이후 새로 생성되는 이슈부터만 반영된다
- 내용 오류가 발견되면 해당 이슈를 수정하지 말고 **새 이슈를 생성**한다
